## 1. 系统简介

审计助手系统是一款面向审计人员的桌面辅助工具，旨在帮助审计团队高效管理审计项目、访谈记录、风险点，并通过 AI 技术辅助分析访谈内容、识别风险指标、生成审查报告。

### 核心功能

| 功能模块 | 说明 |
|---------|------|
| 审计项目管理 | 创建和管理审计项目，支持常规审计、专项审计、跟踪审计等多种类型 |
| 访谈记录管理 | 记录访谈内容、摘要、关键发现和风险指标 |
| 访谈人员管理 | 管理被访谈人信息 |
| 风险点管理 | 创建和跟踪风险点，支持 5x5 风险矩阵可视化 |
| AI 辅助分析 | 单条访谈的摘要生成、关键发现提取、风险指标提取 |
| AI 整体分析 | 项目级别的审查报告生成和项目摘要生成 |
| 待办事项管理 | 跟踪审计过程中的待办任务 |

## 2. 登录与账户

### 2.1 默认账户

系统首次启动时会自动创建默认管理员账户：

| 字段 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `admin123` |

### 2.2 登录

1. 启动应用后进入登录页面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 首次登录后建议立即修改密码

### 2.3 修改密码

1. 点击右上角用户头像
2. 选择"修改密码"
3. 输入当前密码和新密码
4. 点击"确认修改"

## 3. 仪表盘

- **项目统计**：项目总数、进行中项目数、已完成项目数

## 4. 审计项目管理

### 4.1 创建项目

1. 在左侧菜单点击"审计项目"
2. 点击页面右上角"新建项目"按钮
3. 填写项目信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 项目名称 | 审计项目的名称 | 是 |
| 项目类型 | 常规审计/专项审计/跟踪审计/其他 | 是 |
| 审计范围 | 重点审计方向和范围描述 | 否 |
| 被审计单位 | 被审计的单位或部门名称 | 否 |
| 开始日期 | 项目开始日期 | 否 |
| 结束日期 | 项目预计结束日期 | 否 |
| 项目负责人 | 项目负责人姓名 | 否 |
| 预算工时 | 预计审计工时（小时） | 否 |
| 备注 | 其他补充信息 | 否 |

4. 点击"创建"按钮

### 4.2 项目详情

点击项目卡片进入项目详情页，包含以下标签页：

| 标签页 | 说明 |
|--------|------|
| 基本信息 | 查看和编辑项目基本信息 |
| 访谈记录 | 管理该项目下的所有访谈记录 |
| 访谈人员 | 管理所有被访谈人（全局共享） |
| 风险点 | 管理该项目下的风险点 |
| AI 分析 | AI 整体分析和分析结果查看 |
| 待办事项 | 管理该项目下的待办任务 |

## 5. 访谈记录管理

### 5.1 创建访谈记录

1. 进入项目详情页，切换到"访谈记录"标签页
2. 点击"新建访谈"按钮
3. 填写访谈信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 访谈日期 | 访谈发生的日期 | 是 |
| 访谈主题 | 本次访谈的主题 | 是 |
| 被访谈人 | 从已有人员列表中选择 | 是 |
| 访谈人 | 执行访谈的人员 | 否 |
| 开始/结束时间 | 访谈的时间段 | 否 |
| 地点 | 访谈地点 | 否 |
| 访谈内容 | 访谈的详细记录（支持多行文本） | 否 |
| 访谈摘要 | 访谈的核心内容概括 | 否 |
| 关键发现 | 访谈中识别到的重要问题 | 否 |
| 风险指标 | 访谈中发现的潜在风险 | 否 |
| 行动项 | 后续需要执行的行动 | 否 |
| 备注 | 其他补充信息 | 否 |

4. 点击"保存"按钮

### 5.2 编辑访谈记录

1. 在访谈记录列表中点击目标记录的标题或"查看"按钮
2. 进入访谈详情页，修改相关字段
3. 修改完成后点击"保存修改"按钮

> **提示**：修改任何字段后，页面会标记为"未保存"状态，请及时保存。

### 5.3 访谈状态管理

访谈记录支持以下状态流转：

| 状态 | 说明 |
|------|------|
| 草稿 | 访谈记录初始状态，内容可能不完整 |
| 已审查 | 访谈内容已经过审查确认 |
| 已确认 | 访谈记录已最终确认 |
| 已归档 | 访谈记录已归档，不再编辑 |

在访谈详情页右上角，通过状态下拉菜单切换状态。

### 5.4 AI 辅助分析（单条访谈）

在访谈详情页中，系统提供三个 AI 辅助功能按钮，位于对应区域的标题栏右侧：

#### AI 生成摘要

1. 在"访谈摘要"区域，点击 **AI生成摘要** 按钮
2. 系统会根据访谈内容自动生成简洁摘要（不超过300字）
3. 生成的摘要会自动填入摘要文本框
4. 可以手动修改后保存

> **前提**：需要先填写访谈内容。

#### AI 提取关键发现

1. 在"关键发现"区域，点击 **AI提取关键发现** 按钮
2. 系统会从访谈内容中提取：
   - 关键信息和重要陈述
   - **语言矛盾点**（被访谈人前后表述不一致之处）
   - 含糊表述（如"大概""可能""不太清楚"等）
   - 需要进一步核实的事项
3. 提取结果会自动填入关键发现文本框

#### AI 提取风险指标

1. 在"风险指标"区域，点击 **AI提取风险指标** 按钮
2. 系统会识别访谈中的潜在风险，并按等级分类：
   - 严重风险
   - 高风险
   - 中风险
   - 低风险
3. 每个风险会标注来源和可能影响

> **注意**：使用 AI 辅助功能前，请确保已在"AI设置"中配置了有效的 API Key。

## 7. 风险点管理

### 7.1 创建风险点

1. 进入项目详情页，切换到"风险点"标签页
2. 点击"新建风险点"按钮
3. 填写风险信息：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 风险标题 | 风险点的简要标题 | 是 |
| 风险描述 | 详细描述风险内容 | 否 |
| 风险等级 | 低/中/高/严重 | 是 |
| 可能性 | 罕见/不太可能/可能/很可能/几乎确定 | 是 |
| 影响程度 | 可忽略/轻微/中等/重大/灾难性 | 是 |
| 风险分类 | 财务风险/运营风险/合规风险/战略风险/信息风险/其他 | 是 |
| 状态 | 已识别/分析中/缓解中/已解决/已接受/已转移 | 是 |
| 来源 | 风险发现的来源 | 否 |
| 证据 | 相关证据描述 | 否 |
| 缓解方案 | 建议的风险应对措施 | 否 |
| 责任人 | 风险应对责任人 | 否 |
| 截止日期 | 风险应对截止日期 | 否 |

4. 点击"创建风险点"按钮

### 7.2 风险评估参数说明

#### 可能性等级

| 等级 | 说明 | 对应值 |
|------|------|--------|
| 罕见 | 极少发生，仅在特殊情况下可能发生 | rare |
| 不太可能 | 发生的可能性较低 | unlikely |
| 可能 | 在正常情况下可能发生 | possible |
| 很可能 | 较高概率发生 | likely |
| 几乎确定 | 几乎一定会发生 | almost_certain |

#### 影响程度等级

| 等级 | 说明 | 对应值 |
|------|------|--------|
| 可忽略 | 对审计目标影响极小 | insignificant |
| 轻微 | 对审计目标有轻微影响 | minor |
| 中等 | 对审计目标有中等程度影响 | moderate |
| 重大 | 对审计目标有显著影响 | major |
| 灾难性 | 对审计目标有极其严重的影响 | catastrophic |

### 7.3 风险矩阵

系统提供 5x5 风险矩阵可视化视图：

1. 在左侧菜单点击"风险矩阵"
2. 矩阵横轴为"可能性"（从左到右：罕见 → 几乎确定）
3. 矩阵纵轴为"影响程度"（从上到下：灾难性 → 可忽略）
4. 每个单元格的颜色表示风险等级：
   - 绿色 = 低风险
   - 黄色 = 中风险
   - 橙色 = 高风险
   - 红色 = 严重风险
5. 点击有数据的单元格可以查看该风险点的详细信息

## 8. AI 智能分析（项目整体）

### 8.1 生成 AI 审查报告

1. 进入项目详情页，切换到"AI 分析"标签页
2. 点击 **生成AI审查报告** 按钮
3. 系统会综合该项目下所有访谈记录，生成包含以下内容的完整审查报告：
   - **一、关键发现**：从访谈中提取的核心信息
   - **二、风险分析**：逐项分析识别的风险点及风险等级
   - **三、内部控制评估**：相关内部控制的有效性评估
   - **四、信息缺口与待确认事项**：需要进一步核实的信息
   - **五、审计建议**：具体的后续审计行动建议
   - **六、后续沟通方向**：建议下次访谈的重点方向
4. 生成完成后，报告会自动保存并显示在分析列表中

> **前提**：项目中需要已有访谈记录，且已配置 AI API Key。

### 8.2 生成项目摘要

1. 在"AI 分析"标签页，点击 **生成项目摘要** 按钮
2. 系统会根据项目信息和访谈发现生成项目整体摘要
3. 生成完成后自动保存到分析列表

### 8.3 查看 AI 分析结果

1. 在"AI 分析"标签页的分析列表中，点击目标记录的"查看"按钮
2. 弹出对话框显示完整的分析内容
3. 可以复制内容用于报告编写
4. 点击"删除"可以移除不需要的分析记录

## 10. AI 服务配置

### 10.1 配置步骤

1. 在左侧菜单点击"AI 设置"
2. 选择 AI 服务提供商（DeepSeek / 通义千问 / 腾讯元宝 / 豆包）
3. 输入对应的 API Key
4. 选择要使用的模型
5. 点击"保存 API Key"按钮
6. 点击"测试连接"按钮验证配置

### 10.2 获取 API Key

| 提供商 | 获取地址 |
|--------|---------|
| DeepSeek | https://platform.deepseek.com/api_keys |
| 通义千问 | https://dashscope.console.aliyun.com/apiKey |
| 腾讯元宝 | https://console.cloud.tencent.com/hunyuan |
| 豆包 | https://console.volcengine.com/ark |

### 10.3 安全说明

- API Key 通过 AES-256-CBC 加密存储在本地数据库中
- API Key 仅在本地使用，不会上传到第三方服务器
- 所有 AI 请求直接发送到所选提供商的 API 服务器

## 12. 数据备份与恢复

### 12.1 创建备份

1. 进入"系统设置"页面
2. 切换到"数据备份"选项卡
3. 点击"创建备份"按钮
4. 系统会自动在应用数据目录的 `backups/` 子目录下创建备份文件

### 12.2 恢复数据

1. 在备份列表中找到目标备份
2. 点击"恢复"按钮
3. 系统会先自动创建当前数据库的备份（防止误操作）
4. 确认后系统会用备份文件替换当前数据库
5. 恢复完成后需要重启应用

### 12.3 手动备份数据库文件

数据库文件位置：

- **Windows**：按 `Win+R`，输入 `%APPDATA%\audit-assistant-system`
- **macOS**：`~/Library/Application Support/audit-assistant-system/`
- **Linux**：`~/.config/audit-assistant-system/`

复制该目录下的 `.db` 文件即可完成手动备份。

## 13. 常见问题

### Q1：AI 功能提示"请先配置 API Key"

**A**：请进入左侧菜单"AI 设置"页面，选择 AI 服务商并输入 API Key，然后点击"测试连接"确认可用。

### Q2：AI 分析请求超时

**A**：可能原因：
- 网络连接不稳定
- 访谈内容较多，AI 处理时间较长
- AI 服务商服务器响应慢

请检查网络连接后重试。

### Q3：创建风险点失败

**A**：请确保所有下拉选择字段（风险等级、可能性、影响程度、风险分类、状态）都已选择有效值，不能留空。

### Q4：编辑访谈记录后保存失败

**A**：如果出现数据库错误，可能需要重置数据库：
1. 关闭应用
2. 按 `Win+R` 输入 `%APPDATA%\audit-assistant-system`
3. 删除所有 `.db` 文件
4. 重新启动应用

### Q5：AI 分析列表为空

**A**：请确认：
1. 项目中已有访谈记录
2. AI API Key 已正确配置
3. 生成报告/摘要后等待加载完成

### Q6：如何重置数据库

**A**：
1. 关闭应用
2. 打开数据库目录：`Win+R` → `%APPDATA%\audit-assistant-system`
3. 删除所有 `.db`、`.db-wal`、`.db-shm` 文件
4. 重新启动应用，系统会自动重建

> **警告**：重置数据库会清除所有数据，请先备份。

### Q7：忘记管理员密码

**A**：重置数据库后系统会重新创建默认管理员账户（admin/admin123），详见 Q6。
